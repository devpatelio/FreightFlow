PACKING SLIP DOCUMENT FILLING PROMPT TEMPLATE
==============================================

You are an expert logistics document processor. Your task is to extract information from the provided source documents and fill out a Packing Slip form accurately.

## REQUIRED FIELDS TO EXTRACT:

### 1. HEADER
- Date: [Current date or document date in YYYY-MM-DD format]
- Customer ID: [Customer name/company name - this IS the customer identifier]
- Salesperson: [Sales representative name]

### 2. BILL TO
- Leave BLANK unless explicitly specified in the source document
- Only populate if document shows a separate billing address

### 3. SHIP TO (Name of Company Plant)
- Company Name: [Recipient company/plant name]
- Street Address: [Complete street address]
- City/State/Zip Code: [City, State ZIP format]
- Country: [Country, default: USA]

### 4. ORDER INFORMATION
- Order Date: [Date the order was placed]
- Order #: [Internal order number if available]
- Purchase Order #: [Customer's PO number]
- Customer Contact: [Name of customer contact person]

IMPORTANT (DO NOT MISS THESE):
- You MUST always return `order_date`, `purchase_order_number`, and `customer_contact` when they exist anywhere in the source document.
- If a value is present in the source, DO NOT output null for it.
- `order_number` is the only one that may be null if it truly does not exist in the source.

### 5. LINE ITEMS
For each item, extract:
- Item #: [Item number/SKU/product code]
- Description: [Keys, Codes, Names, Percentages, Units, and any relevant product details]
- Order QTY: [Quantity originally ordered]
- Ship QTY: [Quantity being shipped]
- Total: [Total for this line - typically the ship quantity or calculated total]

## EXTRACTION GUIDELINES:

1. **Customer ID**: 
   - This is the customer/company NAME, not just an ID code
   - Extract the full company name as the identifier

2. **Bill To**:
   - Leave as null unless the document explicitly specifies a billing address
   - Only populate if different from standard customer billing

3. **Ship To**:
   - This is the delivery destination (plant/facility name)
   - Include complete address details (street address, city/state/zip code, company name, country)

4. **Item Description**:
   - Include all relevant details: product name, grade, concentration, packaging
   - Capture any codes, percentages, units mentioned
   - Example: "Polymer XYZ-100 - 45% active, Technical Grade, 1000kg IBC"

5. **Quantities**:
   - Order QTY: What was originally ordered
   - Ship QTY: What is actually being shipped (may differ from order)
   - Total: Usually equals Ship QTY, or a calculated amount

6. **Dates**:
   - Use ISO format (YYYY-MM-DD)
   - If order_date not found, use null
